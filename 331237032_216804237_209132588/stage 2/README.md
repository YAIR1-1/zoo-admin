פרויקט ניהול פרויקטים וביקורות - שלב ב': אופטימיזציה, אילוצים וטרנזקציות
מגיש: יאיר אליהו ועמיתי

תיאור הפרויקט: מערכת לניהול פרויקטים, מעקב אחר מדדי ביצוע (KPI), ניהול משימות וביצוע ביקורות מחלקתיות. שלב זה מתמקד בשיפור ביצועי המערכת, אבטחת תקינות הנתונים והדגמת אמינות מסד הנתונים.

תוכן הקבצים המצורפים
1. Queries.sql
קובץ המכיל שאילתות תפעוליות מתקדמות:

UPDATE: שלוש שאילתות לעדכון נתונים רוחבי (קידום מנהלים, סגירת משימות אוטומטית ועדכון תקציבים).

DELETE: שלוש שאילתות למחיקת נתונים מבוססת תנאים (ניקוי דוחות ישנים, הסרת ביקורות סגורות ומחיקת משימות פגות תוקף).

כל שאילתה מלווה בבדיקת SELECT מקדימה לצורך אימות הנתונים.

2. Constraints.sql
קובץ המגדיר חוקים עסקיים ברמת מסד הנתונים:

אילוץ CHECK המבטיח שתאריך סיום פרויקט יהיה מאוחר מתאריך ההתחלה.

אילוץ CHECK המונע הזנת ערכי KPI שליליים.

אילוץ UNIQUE המבטיח שמות פרויקטים ייחודיים במערכת.

הקובץ כולל פקודות INSERT שגויות בכוונה להוכחת פעילות האילוצים.

3. Index.sql
קובץ המטפל באופטימיזציה ושיפור זמני ריצה:

יצירת אינדקסים על עמודות תאריך, מפתחות זרים ושמות מחלקות בטבלת ה-Audit (המכילה 20,000 רשומות).

הדגמת השיפור בביצועים בעזרת פקודת EXPLAIN ANALYZE (מעבר מ-Sequential Scan ל-Index Scan).

4. RollbackCommit.sql
הדגמת מנגנון הטרנזקציות (ACID) במסד הנתונים:

ROLLBACK: ביצוע עדכון וביטולו כדי להחזיר את המצב לקדמותו.

COMMIT: ביצוע עדכון ושמירתו הקבועה במסד הנתונים.

הוראות הרצה
יש לטעון תחילה את בסיס הנתונים מהגיבוי המקורי (backup_13.4.sql).
